## 📊 探校之旅数据库表定义文档

### 一、表关系概览

```
┌─────────┐          ┌──────────────┐          ┌──────────┐
│  users  │ 1───n   │  save_slots  │ n───1   │ stories  │
│  用户表  │────────▶│   存档表      │─────────▶│  剧情主表 │
└─────────┘          └──────────────┘          └──────────┘
     │                                                   │
     │ 1                                                 │ 1
     │                                                   │
     │ n                                                 │ n
     ▼                                                   ▼
┌──────────────┐          ┌──────────────┐    ┌──────────────┐
│ user_badges  │ n───1   │   badges     │    │ story_nodes  │
│ 用户勋章关系 │─────────▶│   勋章定义表  │    │  剧情节点表   │
└──────────────┘          └──────────────┘    └──────────────┘
```

**核心关系说明**：
- 一个用户 → 多个存档、多个勋章
- 一个剧情 → 多个节点、多个存档
- 一个勋章 → 可以被多个用户获得

---

### 二、表详细定义

#### 1. 用户表 (`users`)

**用途**：存储所有普通用户的基本信息。

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | INT | **主键**，自增 | 用户唯一编号 |
| `openid` | VARCHAR(64) | **唯一**，非空 | 微信用户唯一标识（用于登录） |
| `name` | VARCHAR(50) | 非空，默认空串 | 用户姓名 |
| `gender` | VARCHAR(10) | 非空，默认空串 | 性别（男/女/其他） |
| `grade` | VARCHAR(20) | 可为空 | 年级（如 2026级） |
| `college` | VARCHAR(100) | 可为空 | 所属学院 |
| `major` | VARCHAR(100) | 可为空 | 所学专业 |
| `register_time` | DATETIME | 默认当前时间 | 注册时间 |
| `update_time` | DATETIME | 自动更新 | 资料更新时间 |

---

#### 2. 管理员表 (`admins`)

**用途**：存储管理员账号信息，用于后台登录。

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | INT | **主键**，自增 | 管理员唯一编号 |
| `username` | VARCHAR(50) | **唯一**，非空 | 管理员登录账号 |
| `password_hash` | VARCHAR(128) | 非空 | 密码加密后的哈希值（bcrypt） |
| `failed_count` | INT | 默认 0 | 连续登录失败次数 |
| `locked_until` | DATETIME | 可为空 | 账号锁定截止时间（空表示未锁定） |
| `created_at` | DATETIME | 默认当前时间 | 账号创建时间 |

---

#### 3. 剧情主表 (`stories`)

**用途**：定义所有剧情（长故事/短故事）的基本信息。

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | INT | **主键**，自增 | 剧情唯一编号 |
| `code` | VARCHAR(50) | **唯一**，非空 | 剧情编码（如 `campus`、`medical`） |
| `name` | VARCHAR(100) | 非空 | 剧情名称（如“校园巡礼”） |
| `type` | VARCHAR(20) | 非空 | 类型：`long`（长故事）/ `short`（短故事） |
| `description` | TEXT | 可为空 | 剧情简介 |
| `badge_id` | INT | 可为空 | 关联的勋章ID（完成剧情后获得） |
| `max_saves` | INT | 默认 5 | 最大存档数量（仅长故事有效） |
| `status` | VARCHAR(20) | 默认 `enabled` | 状态：`enabled`（启用）/ `disabled`（停用） |

**前端重点关注**：`type` 决定是否显示存档按钮，`badge_id` 决定完成后发哪个勋章。

---

#### 4. 剧情节点表 (`story_nodes`)

**用途**：存储每个剧情的具体对话/场景内容（按顺序播放）。

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | INT | **主键**，自增 | 节点唯一编号 |
| `story_id` | INT | **外键**，非空 | 所属剧情ID（关联 `stories.id`） |
| `dialogue_text` | TEXT | 非空 | 对话/旁白文字内容 |
| `speaker` | VARCHAR(50) | 可为空 | 说话人名称（如“学姐”“系统”） |
| `bg_image_url` | VARCHAR(255) | 可为空 | 背景图片URL |
| `is_end` | BOOLEAN | 默认 `FALSE` | 是否为剧情结束节点 |
| `sort_order` | INT | 非空 | 播放顺序（1, 2, 3…） |

**前端重点关注**：按 `sort_order` 从小到大播放，当 `is_end = true` 时触发完成剧情逻辑。

---

#### 5. 勋章定义表 (`badges`)

**用途**：定义所有可获得的勋章信息。

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | INT | **主键**，自增 | 勋章唯一编号 |
| `code` | VARCHAR(50) | **唯一**，非空 | 勋章编码（如 `campus_explorer`） |
| `name` | VARCHAR(50) | 非空 | 勋章名称（如“校园探索者”） |
| `icon_url` | VARCHAR(255) | 非空 | 勋章图标URL（彩色） |
| `description` | VARCHAR(255) | 可为空 | 勋章描述/达成条件说明 |

---

#### 6. 用户勋章关系表 (`user_badges`)

**用途**：记录每个用户已获得的勋章（多对多关系）。

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `user_id` | INT | **联合主键**，外键 | 用户ID（关联 `users.id`） |
| `badge_id` | INT | **联合主键**，外键 | 勋章ID（关联 `badges.id`） |
| `story_id` | INT | 非空 | 通过完成哪个剧情获得的 |
| `obtained_time` | DATETIME | 默认当前时间 | 获得勋章的时间 |

**前端重点关注**：查询此表判断勋章是否已点亮（彩色/灰色）。

---

#### 7. 存档表 (`save_slots`)

**用途**：存储用户在长故事中保存的进度（最多5个档位）。

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `slot_id` | INT | **主键**，自增 | 存档记录唯一编号 |
| `user_id` | INT | **外键**，非空 | 用户ID（关联 `users.id`） |
| `story_id` | INT | **外键**，非空 | 所属剧情ID（关联 `stories.id`） |
| `node_id` | INT | **外键**，非空 | 当前所在的节点ID（关联 `story_nodes.id`） |
| `slot_index` | INT | **联合唯一**，非空 | 档位编号（1~5） |
| `save_time` | DATETIME | 默认当前时间 | 存档保存时间 |

**唯一约束**：`(user_id, story_id, slot_index)` 联合唯一，确保一个用户对一个剧情的同一个档位只能存一次。

---

### 三、快速索引（按功能查找表）

| 要查什么 | 去哪个表 |
| :--- | :--- |
| 用户个人信息 | `users` |
| 管理员账号密码 | `admins` |
| 所有剧情列表 | `stories` |
| 某个剧情的对话内容 | `story_nodes`（按 `story_id` 过滤） |
| 所有勋章定义 | `badges` |
| 某个用户已获得的勋章 | `user_badges`（按 `user_id` 过滤） |
| 某个用户的存档列表 | `save_slots`（按 `user_id` 过滤） |

### 四、数据类型速查（给前端同学看）

| 前端类型 | 数据库类型 | 示例 |
| :--- | :--- | :--- |
| 整数 | `INT` | `id: 1` |
| 字符串 | `VARCHAR` / `TEXT` | `name: "张三"` |
| 布尔值 | `BOOLEAN`（存 0/1） | `is_end: false` |
| 时间 | `DATETIME` | `"2026-07-07 10:30:00"` |
| 可为空 | 字段后无 `NOT NULL` | 接口返回可能为 `null` |

---

### 五、给任晟达的额外说明

1. **外键约束**：`story_nodes`、`save_slots`、`user_badges` 都带有外键，删除 `stories` 或 `users` 时会自动删除关联数据（`ON DELETE CASCADE`）。
2. **存档数限制**：`stories.max_saves` 字段定义了最大存档数（长故事默认5），短故事不应显示存档入口。
3. **勋章幂等性**：`user_badges` 使用 `(user_id, badge_id)` 作为联合主键，天然防止重复插入同一勋章。
