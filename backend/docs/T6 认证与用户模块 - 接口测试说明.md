1. 通用约定
项目	说明
基地址	http://localhost:3000
认证失败 HTTP 状态码	401 Unauthorized（已按 Q-21 实现）
越权访问 HTTP 状态码	403 Forbidden
普通业务错误 HTTP 状态码	200 OK（业务结果看 code）
统一响应格式	{ code: 0, message: "success", data: ... }
2. 错误码（T6 相关）
code	HTTP	含义
0	200	成功
1001	200	微信登录失败
1002	401	缺少认证 token
1003	401	token 无效或已过期
1004	403	无权操作该资源（越权）
2001	200	缺少必要参数
2002	200	参数非法
3001	200	管理员用户名或密码错误
3002	200	管理员账号已锁定（15 分钟）
4001	200	用户不存在
4002	200	管理员不存在
5000	200	服务器内部错误

4.2 GET /api/users/me —— 获取当前用户信息

功能：获取当前登录用户的个人资料（不返回 openid）。
✅ 正确运行

bash

curl -X GET http://localhost:3000/api/users/me \

❌ 错误运行
场景	操作	预期 HTTP	预期 code	预期 message
未携带 token	不带 Authorization 头	401	1002	缺少认证token
token 无效/过期	Bearer invalid_token	401	1003	无效或过期的token
用户不存在	用已被删除的 userId 签发 token	401	4001	用户不存在
bash

# 测试未携带 token
curl -X GET http://localhost:3000/api/users/me

# 测试无效 token
curl -X GET http://localhost:3000/api/users/me \
  -H "Authorization: Bearer invalid_token"

4.4 POST /api/admin/login —— 管理员登录

功能：管理员通过用户名和密码登录，连续 5 次失败锁定 15 分钟。
✅ 正确运行
项目	内容
操作	{"username":"admin","password":"rjks@bjut514"}
预期 HTTP	200 OK
预期响应	{"code":0,"message":"success","data":{"token":"..."}}
说明	返回 adminToken，有效期 2 小时
bash

curl -X POST http://localhost:3000/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"rjks@bjut514"}'

❌ 错误运行
场景	操作	预期 HTTP	预期 code	预期 message
用户名不存在	{"username":"wrong","password":"xxx"}	200	3001	用户名或密码错误
密码错误（第 1-4 次）	{"username":"admin","password":"wrong"}	200	3001	用户名或密码错误，还剩 X 次机会
密码错误（第 5 次）	连续 5 次错误	200	3002	连续失败5次，账号已锁定15分钟
账号已锁定	锁定期间再次登录	200	3002	账号已被锁定，请于 ... 后重试
缺少用户名	{"password":"xxx"}	200	2001	用户名和密码不能为空
缺少密码	{"username":"admin"}	200	2001	用户名和密码不能为空
bash

# 测试密码错误
curl -X POST http://localhost:3000/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"wrong_password"}'

# 解锁管理员（锁定后执行）
curl -X PUT http://localhost:3000/api/admin/unlock \
  -H "Authorization: Bearer <adminToken>" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin"}'

    ⚠️ 解锁接口需要额外实现，测试时可直接在数据库执行：
    sql

    UPDATE admins SET failed_count = 0, locked_until = NULL WHERE username = 'admin';

4.6 越权与鉴权验证
✅ 垂直越权防护
场景	操作	预期 HTTP	预期 code
普通用户调后台接口	GET /api/admin/stats 带 userToken	401	1003
管理员调用户接口	GET /api/users/me 带 adminToken	401	1003
bash

# 普通用户调后台接口（应返回 401）
curl -X GET http://localhost:3000/api/admin/stats \
  -H "Authorization: Bearer <userToken>"

# 管理员调用户接口（应返回 401）
curl -X GET http://localhost:3000/api/users/me \
  -H "Authorization: Bearer <adminToken>"

✅ 水平越权防护（删除他人存档）
场景	操作	预期 HTTP	预期 code
删除他人存档	用用户 B 的 token 删除用户 A 的存档	403	1004
bash

# 先登录用户 A 并创建存档
curl -X POST http://localhost:3000/api/auth/wechat-login -d '{"code":"userA"}'
# 记录 tokenA 和存档 slotId

# 登录用户 B
curl -X POST http://localhost:3000/api/auth/wechat-login -d '{"code":"userB"}'
# 用 tokenB 删除 userA 的存档
curl -X DELETE http://localhost:3000/api/save-slots/<slotId> \
  -H "Authorization: Bearer <tokenB>"
# 预期 HTTP 403 + code 1004
