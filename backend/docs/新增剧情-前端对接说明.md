## 1. 前端要做的（很少）

### ① 大学规划（长剧情）——【不用改，自动出现】
- 故事模式页 `story-mode.js` 是数据驱动的（拉 `getStoryList()` 过滤 `type==='long'`），**加进库就自动出现在列表**，点击进 `campus-story.js` 渲染。
- 分支走 campus-story 现有的 `selectChoice`（按 `targetNodeId` 跳转），已支持，不用改。
- ✅ 你只需打开故事模式，确认"大学规划"卡片出现、能进、分支能选、能返回、能完成发章。

### ② 一卡通补办（短剧情）——【不用改，复用 card】
- 复用了原来的 `card` 剧情（本就是"一卡通补办"占位），已填真实内容并启用；`card` 流程标记已启用。
- `process-map.js` 的 `markerLocationIds` 里**已经有 `card: 'student_service_center'`**，不用动。
- ✅ 打开流程地图，"一卡通补办"标记应出现在学生综合服务楼位置，点进去能走完。

### ③ 选课攻略 / 境外交流（短剧情）——【要改 2 行】
这两个是新短剧情，走流程地图标记进入。**请在 `frontend/pages/process-map/process-map.js` 的 `markerLocationIds` 里加两条**（地点在 `campus-map.js` 里都已存在，不用新画坐标）：

```js
const markerLocationIds = {
  medical: 'school_hospital',
  card: 'student_service_center',
  course_select: 'no3_teaching_building',   // 选课 → 第三教学楼
  overseas_exchange: 'science_building'      // 境外交流 → 科学楼（国际处科学楼221）
};
```

> 不加这两行，这两个标记会因为拿不到地点坐标而定位不准（挤在默认位置）。加完就能在三教/科学楼位置正常显示、点进去走剧情。
> `process-select.js`（如果还在用的旧流程选择页）如有同类硬编码表，一并加。

### ④ 三枚新徽章图标（王子涵，素材）——【可后补，不阻塞剧情】
新徽章 `badge_course_select` / `badge_overseas` / `badge_plan` 的 icon 指向：
- `/images/badge_course_select.png`
- `/images/badge_overseas.png`
- `/images/badge_plan.png`

请在 `frontend/images/` 放这三张 PNG（风格同现有徽章图）。**在补上之前，勋章墙上这三枚会显示为裂图**——剧情、发章、完成都不受影响，只是图标缺失。急的话可先临时复用一张现有徽章图占位。
