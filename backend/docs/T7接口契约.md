## 一、通用约定

### 1.1 基地址

| 环境 | 地址 |
| --- | --- |
| 本地联调 | `http://localhost:3000` |
| 真机预览 | ngrok 的 https 域名（需加入小程序后台白名单，见 §5） |

### 1.2 鉴权

所有接口都要带请求头：

```
Authorization: Bearer <token>
```

- **用户接口**用 `wxLogin` 拿到的 `token`
- **`/api/admin/*` 接口**用管理员登录拿到的 `adminToken`

两种 token 的 payload 里带 `type` 字段（`user` / `admin`），中间件会校验。**普通用户的 token 调后台接口会被拒**，返回 `1003`。

> ⚠️ **前端不要再自己传 userId。** 后端从 token 里解出 `userId`，「我的存档 / 我的勋章」一律按它过滤。现有代码里的 `const userId = getApp().globalData.token` 全部可以删掉（详见 §4.1）。

### 1.3 统一响应格式

成功：

```json
{ "code": 0, "message": "success", "data": ... }
```

失败：

```json
{ "code": 3101, "message": "存档已满（最多 5 个），请先删除已有存档", "details": null }
```

**除越权外，失败也返回 HTTP 200**，业务结果看 `code`。这是有意为之：微信小程序的 `wx.request` 遇到非 2xx 不走 `fail` 回调，用 `200 + 业务码` 前端更好处理。

**唯一的例外**：越权访问他人资源返回真实的 **HTTP 403**（`code: 1004`）。这是测试文档 FN-11-08 / BR-26 / RSK-03 的硬性要求。

> Q-21 仍待架构最终拍板（认证失败要不要一并返 401）。若改，后端只动 `response.js`，前端只需多判一个 `statusCode`。

### 1.4 错误码

分段规则：`1000` 认证与权限 / `2000` 参数 / `3000` 业务规则 / `4000` 资源不存在 / `5000+` 服务端。
其中 `3001-3099`、`4001-4099` 归后端 A，`3100+`、`4100+` 归后端 B。

| code | HTTP | 含义 | 前端建议 |
| --- | --- | --- | --- |
| `0` | 200 | 成功 | — |
| `1002` | 200 | 缺少认证 token | 跳登录页 |
| `1003` | 200 | token 无效或已过期 | 清除 token，跳登录页 / 管理员登录页 |
| `1004` | **403** | 无权操作该资源 | Toast「无权操作该存档」并刷新列表 |
| `2001` | 200 | 缺少必要参数 | 一般是前端 bug |
| `2002` | 200 | 参数非法 | 同上 |
| `3001` | 200 | 管理员用户名或密码错误 | 红字提示 |
| `3002` | 200 | 管理员账号已锁定（15 分钟） | 提示剩余时间 |
| `3101` | 200 | 存档已满 | 弹窗「最多保存5个存档，请先删除旧存档」 |
| `3102` | 200 | 短剧情不支持存档 | 短剧情页不该出现存档按钮 |
| `4101` | 200 | 剧情不存在或暂未开放 | Toast「加载剧情失败，请重试」 |
| `4102` | 200 | 剧情节点不存在 | 存档数据异常，建议删除 |
| `4103` | 200 | 存档不存在 | 刷新列表 |
| `4104` | 200 | 地图资源未配置 | 提示「地图资源更新中，请稍后查看」 |
| `4105` | 200 | 暂无可导出数据 | 提示「暂无用户数据可导出」，**后端不生成空文件** |
| `5000` | 200 | 服务器内部错误 | Toast「网络连接失败，请稍后重试」 |

## 二、用户端接口（7 个）

### 2.1 `GET /api/stories` —— 剧情列表

可选 query：`type=long` / `type=short`

`completed` 按当前用户计算。`status=disabled` 的剧情不下发（V1.0 的「一卡通补办」「打印流程」）。

```json
{
  "code": 0, "message": "success",
  "data": [
    { "id": "campus",  "name": "浏览校园",     "type": "long",  "typeLabel": "长故事",
      "description": "跟随向导学姐了解校园各建筑与场所",
      "maxSaves": 5, "supportsSave": true,  "completed": false },
    { "id": "medical", "name": "医保报销流程", "type": "short", "typeLabel": "短故事",
      "description": "了解校医院就诊与医保报销的完整流程",
      "maxSaves": 0, "supportsSave": false, "completed": true }
  ]
}
```

### 2.2 `GET /api/stories/:id/nodes` —— 剧情节点

`:id` 传剧情 code（`campus` / `medical`），也支持自增 id。
可选 query：`fromNodeId=n4` —— 只返回该节点及之后的部分，用于读档续读。

**字段名与现有 `utils/api.js` 完全一致，前端渲染逻辑一行不用改。**

```json
{
  "code": 0, "message": "success",
  "data": [
    { "id": "n1", "bg": "/images/story_bg1.jpg", "character": "学姐",
      "characterImage": "/images/character_xuejie.png",
      "text": "欢迎来到北京工业大学！我是你的向导学姐，今天带你逛逛校园。",
      "isEnd": false, "sortOrder": 1 },
    "...",
    { "id": "n_end", "bg": "/images/story_bg_end.jpg", "character": "学姐",
      "characterImage": "/images/character_xuejie.png",
      "text": "🎉 校园就介绍到这里啦！\n\n点击完成剧情，获得勋章！",
      "isEnd": true, "sortOrder": 6 }
  ]
}
```

> **`id` 一定是字符串 `node_key`（`n1` / `n_end`），不是数字。**
> 原因：`campus-story.js` 读档时执行 `nodes.findIndex(n => n.id === startNodeId)`，而 `startNodeId` 来自 URL 参数永远是字符串。若返回数字 `3`，`'3' === 3` 为 `false`，读档会静默失败、每次都从第一句重播。
> 同理 `saves.js` 用 `parseInt(nodeId.replace(/\D/g,''))` 算存档进度，所以 `node_key` 必须含阿拉伯数字。

### 2.3 `POST /api/stories/:id/complete` —— 完成剧情并发放勋章

无请求体。

**幂等**：重复完成不会重复发放，返回 `alreadyObtained: true`。
并发快速连点也安全——「记录完成状态 + 插入勋章」在同一事务里，靠 `user_badges (user_id, badge_id)` 联合主键去重。

首次完成：

```json
{
  "code": 0, "message": "success",
  "data": {
    "storyId": "campus", "completed": true,
    "badge": { "id": "badge_campus", "name": "🏛️ 校园探索者",
               "icon": "/images/badge_campus.png",
               "description": "完成浏览校园剧情，熟悉校园环境", "obtained": true },
    "alreadyObtained": false
  }
}
```

重复完成时结构相同，只有 `"alreadyObtained": true`。

### 2.4 `GET /api/badges/me` —— 勋章墙

返回**全部 4 枚**，未获得的 `obtained: false`，前端据此渲染灰色。

```json
{
  "code": 0, "message": "success",
  "data": [
    { "id": "badge_campus",  "name": "🏛️ 校园探索者", "icon": "/images/badge_campus.png",
      "description": "完成浏览校园剧情，熟悉校园环境", "obtained": true },
    { "id": "badge_medical", "name": "🏥 医疗达人",   "icon": "/images/badge_medical.png",
      "description": "完成医保报销剧情，了解就医流程", "obtained": false },
    { "id": "badge_card",    "name": "💳 补办高手",   "icon": "/images/badge_card.png",
      "description": "完成一卡通补办剧情，了解挂失流程", "obtained": false },
    { "id": "badge_print",   "name": "🖨️ 打印能手",   "icon": "/images/badge_print.png",
      "description": "完成打印流程剧情，了解打印服务", "obtained": false }
  ]
}
```

> `obtainedCount` 仍由前端 `badges.filter(b => b.obtained).length` 计算，后端不返回。

### 2.5 存档

#### `GET /api/save-slots` —— 存档列表

可选 query：`storyId=campus`（**不传就返回该用户全部存档**，前端 `saves.js` 就是不传）。
按存档时间**倒序**（FN-11-01）。

```json
{
  "code": 0, "message": "success",
  "data": [
    { "slotId": 13, "slotIndex": 1,
      "storyId": "campus", "storyName": "浏览校园",
      "nodeId": "n3", "nodeIndex": 3,
      "nodeSummary": "那边是图书馆，你可以在这里自习、借书，记",
      "saveTime": "2026-07-09T16:02:16.000Z" }
  ]
}
```

`storyName` / `nodeIndex` / `nodeSummary` 后端已经算好了（待澄清事项 Q-08 的结论：存档摘要须含存档时间、保存结点、对应剧情名称）。**前端可以删掉 `saves.js` 里写死的 `storyName: '浏览校园'` 和自己 `parseInt` 抠数字的逻辑。**

`saveTime` 是 ISO 字符串，`new Date()` 直接能解析。

#### `POST /api/save-slots` —— 保存进度

```json
{ "storyId": "campus", "nodeId": "n3" }
```

`slotIndex` **可选**。不传时后端自动分配 **1~5 中最小的空闲档位**；删掉档位 3 之后再存，会重新占用档位 3。传了则覆盖指定档位（为后续迭代留的口子，V1.0 前端不用传）。

成功：

```json
{ "code": 0, "message": "success",
  "data": { "slotId": 13, "storyId": "campus", "storyName": "浏览校园",
            "nodeId": "n3", "slotIndex": 1 } }
```

两条业务规则由**后端强制**，不依赖前端隐藏按钮（请求可以绕过前端直接发）：

| 情况 | code | message |
| --- | --- | --- |
| 已有 5 个存档 | `3101` | 存档已满（最多 5 个），请先删除已有存档 |
| 对短剧情存档 | `3102` | 短剧情不支持存档 |
| 节点不属于该剧情 | `4102` | 该剧情下不存在此节点 |

#### `DELETE /api/save-slots/:slotId` —— 删除存档

`:slotId` 是列表返回的数字 `slotId`。

- 成功：`{ "code": 0, "data": { "deleted": true, "slotId": 13 } }`
- 存档不存在：`code 4103`
- **存档不属于当前用户：HTTP 403 + `code 1004`**

> 后端按 token 里的 `userId` 校验归属。前端不需要（也不应该）再传 userId。

### 2.6 `GET /api/process-markers` —— 办事流程标记

`completed` 按当前用户计算：同一批标记，各人看到的勾 / 感叹号不同。
V1.0 **只返回 `medical` 一个标记**，`card` / `print` 已按待澄清事项 Q-09 的结论隐藏。

```json
{
  "code": 0, "message": "success",
  "data": [
    { "id": "medical", "name": "🏥 医保报销流程",
      "description": "了解校医院就诊和医保报销的完整流程",
      "steps": ["在校医院挂号就诊", "医生开具转诊单（如需外院就诊）",
                "持转诊单到指定医院就诊", "收集发票、病历等材料",
                "返校提交报销材料到校医院窗口", "等待审核，报销款打入银行卡"],
      "positionX": 20.5, "positionY": 45.3,
      "storyId": "medical", "completed": true }
  ]
}
```

`id` 就是 `process-select.wxml` 里标记的 `data-id`。`steps` 是弹窗要展示的办理步骤，**前端 `process-select.js` 里那份写死的 `processData` 可以删了**。

`positionX/Y` 是百分比。前端当前用 CSS 定位，暂时可以忽略；将来要数据驱动时用得上。

### 2.7 `GET /api/maps/active` —— 校园地图

```json
{ "code": 0, "message": "success",
  "data": { "id": 1, "name": "平乐园校区平面图",
            "imageUrl": "/images/campus_map_full.png", "version": "v1" } }
```

调用时后端会记一条浏览日志（`user_map_views`），失败不影响返回。
缩放、拖拽、长按保存都是前端和微信能力，后端不参与。
地图未配置时返回 `4104`。

## 三、后台接口（5 个，需 adminToken）

### 3.1 `GET /api/admin/stats` —— 统计卡片

```json
{ "code": 0, "message": "success",
  "data": { "totalUsers": 2, "completedStories": 2, "totalBadges": 2 } }
```

> **`completedStories` 的口径**：待澄清事项 Q-13 已定为「历史累计数，**不包含重复计数**」。
> 一个用户把同一个剧情通关 10 次，只算 1 次。

### 3.2 `GET /api/admin/users?page=1&pageSize=20` —— 用户分页列表

`pageSize` 上限 200。**不返回 `openid`**（NFR-17 最小必要原则）。

```json
{ "code": 0, "message": "success",
  "data": {
    "list": [ { "id": 1, "name": "任晟达", "gender": "男", "grade": "大一",
                "college": "信息学部", "major": "软件工程",
                "registerTime": "2026-07-09T16:02:15.000Z" } ],
    "total": 2, "page": 1, "pageSize": 20 } }
```

> ⚠️ 注意这里和现有 mock 不一样：mock 的 `getAdminStats` 把 `users` 数组塞在 stats 里返回。
> 真实接口按《软件设计说明》4.2 拆成了两个：统计用 `/stats`，列表用 `/users`。前端要调两次。

### 3.3 `GET /api/admin/stories` —— 剧情树预览

返回**全部剧情**（含 `disabled` 的后续迭代剧情）及其节点。

```json
{ "code": 0, "message": "success",
  "data": [
    { "id": "campus", "name": "浏览校园", "type": "长故事", "rawType": "long",
      "status": "enabled", "nodeCount": 6,
      "nodes": [ { "id": "n1", "character": "学姐", "text": "欢迎来到北京工业大学！...",
                   "bg": "/images/story_bg1.jpg",
                   "characterImage": "/images/character_xuejie.png",
                   "isEnd": false, "sortOrder": 1 } ] }
  ] }
```

`type` 直接下发中文标签，因为 `admin-dashboard.wxml` 里判断的是 `item.type === '长故事'`。
`expanded` 是纯前端状态，后端不返回，前端自己 `map` 时加上 `expanded: false` 即可。

### 3.4 导出

| 接口 | Content-Type | 文件名 |
| --- | --- | --- |
| `GET /api/admin/export/users.xlsx` | `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` | `users_20260710_143022.xlsx` |
| `GET /api/admin/export/stories.txt` | `text/plain; charset=utf-8` | `stories_20260710_143022.txt` |

- **无数据时不生成空文件**，返回 `code 4105`（BR-36 / FN-15-05 / FN-16-04）
- Excel 含两个工作表：「统计」（总注册人数、导出时间）和「用户数据」（序号/姓名/性别/年级/学院/专业/注册时间）
- 用户可控字段做了 **Excel 公式注入转义**：以 `= + - @` 开头的值会加 `'` 前缀，防止 `=HYPERLINK(...)` 被 Excel 当公式执行
- TXT 带 **UTF-8 BOM**、换行 **CRLF**，Windows 记事本打开不乱码；剧情文案里自带的换行原样保留
- 每次导出写一条 `operation_logs`（导出人、动作、IP、时间）。**导出人不写进文件名**

## 四、前端需要改的地方

### 4.1 【必改】删掉 `userId = token` 这个 hack

`utils/api.js` 的 `getCurrentUserId()` 直接 `return getApp().globalData.token`，`badges.js` / `saves.js` / `profile.js` 里也各自写了一遍 `const userId = getApp().globalData.token || 'mock_user_id'`。

接真实后端后，**userId 由后端从 token 里解出来**，前端不需要也不应该传。所有这些行都可以删。

推荐把 `utils/api.js` 改成一个统一的请求封装：

```js
const BASE = 'http://localhost:3000';

const request = (method, path, data) => new Promise((resolve, reject) => {
  wx.request({
    url: BASE + path,
    method,
    data,
    header: { Authorization: 'Bearer ' + wx.getStorageSync('token') },
    success: (res) => {
      // 越权是唯一返回非 2xx 的情况
      if (res.statusCode === 403) return reject(res.data);
      if (res.data.code === 0) return resolve(res.data.data);
      if (res.data.code === 1002 || res.data.code === 1003) {
        wx.removeStorageSync('token');
        wx.reLaunch({ url: '/pages/login/login' });
      }
      reject(res.data);   // 带着 code / message 抛出去
    },
    fail: () => reject({ code: 5000, message: '网络连接失败，请稍后重试' })
  });
});

// 后台接口换用 adminToken
const adminRequest = (method, path, data) => /* 同上，header 用 adminToken */;
```

### 4.2 【必改】`admin-dashboard.js` 三处

1. **硬编码覆盖**：`this.setData({ stats: { ...stats, completedStories: 45, totalBadges: 32 } })` —— 这两个数字会把后端返回的真实统计盖掉，删掉。
2. **用户列表**：`stats.users` 不再存在，改调 `GET /api/admin/users`，取 `data.list`。
3. **剧情树**：`storyList` 整段写死在 `data` 里，改调 `GET /api/admin/stories`，`map` 时补上 `expanded: false`。

### 4.3 【必改】`process-select.js`

- `processData` 里写死的 `steps` 改用 `GET /api/process-markers` 返回的 `steps`
- 完成态不再读 `wx.getStorageSync('completedProcesses')`，改用接口返回的 `completed`
  （那个缓存键**没有 userId 后缀**，同一台手机换账号会串数据）
- **`card` 和 `print` 两个标记点要隐藏**（Q-09 已定），接口不会下发它们

### 4.4 【建议改】`saves.js`

`storyName` 和 `nodeIndex` 后端已经算好了，可以删掉写死的 `'浏览校园'` 和 `parseInt` 逻辑。

顺带修一个既有 bug：在 `n_end` 节点存档时，`parseInt('n_end'.replace(/\D/g,''))` 得到 `NaN`，被 `|| 0` 兜成 0，进度显示成 0。改用后端的 `nodeIndex` 就没这问题。

### 4.5 【必改】错误处理改看 `code`

`campus-story.js` 现在判断 `e.message === '存档已满'`，接真接口后要改成判断 `err.code === 3101`。

## 五、微信小程序下载导出文件的注意事项

**小程序不能像浏览器那样自动下载文件，也读不到 `Content-Disposition` 里的 filename。**

正确姿势：

```js
wx.downloadFile({
  url: BASE + '/api/admin/export/users.xlsx',
  header: { Authorization: 'Bearer ' + wx.getStorageSync('adminToken') },
  success(res) {
    if (res.statusCode !== 200) return wx.showToast({ title: '导出失败', icon: 'none' });
    wx.openDocument({ filePath: res.tempFilePath, fileType: 'xlsx' });
    // 或 wx.saveFileToDisk({ filePath: res.tempFilePath })
  }
});
```

文件名后端额外通过 **`X-Filename` 响应头**下发（URL 编码过，用 `decodeURIComponent` 还原），并已配置 `Access-Control-Expose-Headers`。

> ⚠️ **`downloadFile` 合法域名和 `request` 合法域名是小程序后台里两个独立的白名单**，很容易漏配一个。上线前请架构（刘宇宸）两个都加上。

## 六、本地联调步骤

```bash
git checkout feat/t7-story-api
cd backend
npm install
cp .env.example .env        # 填上你本机的 DB_PASSWORD
```

`.env` 里值含 `#` 必须加双引号，否则 `dotenv` 会把 `#` 之后的内容当注释丢掉。

建库并按顺序执行三个脚本：

```sql
CREATE DATABASE IF NOT EXISTS tanxiaozhilv DEFAULT CHARSET utf8mb4;
USE tanxiaozhilv;
```
