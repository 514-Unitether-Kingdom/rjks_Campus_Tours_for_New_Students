# 《浏览校园》v2 后端接入说明

## 结论

前端已支持地点地图、地图页高亮闪烁、选项分支和“专业为空时提示填写”。**目前数据库仍保存旧版 6 个 `campus` 节点，因此必须完成本说明中的数据库和接口改动后，新剧情才会出现在小程序中。**

不要重新执行旧 `t7_seed.sql` 来更新校园剧情：其中使用 `INSERT IGNORE`，已有的旧节点不会被覆盖。

## 1. 数据库字段

新库：使用已更新的 `sql/t7_schema.sql`。

已部署旧库：先备份，再执行 `sql/t9_campus_story_migration.sql`。该文件为一次性迁移；若列已存在，不要重复执行对应 `ALTER TABLE`。

`story_nodes` 新增字段：

| 列 | 类型 | 用途 |
| --- | --- | --- |
| `location_id` | `VARCHAR(64)`，可空 | 地图地点 ID；无明确地点时为 `NULL`。 |
| `node_type` | `scene` / `map` / `choice` | 实景页、地图页、分支选择页。 |
| `choices_json` | `JSON`，可空 | 分支选项数组。仅 `choice` 节点填写。 |

## 2. 接口响应

`GET /api/stories/campus/nodes` 的每个节点必须在原字段基础上增加：

```json
{
  "id": "life_north_dorm_map",
  "text": "看这片闪烁的金色区域……",
  "locationId": "north_dormitories",
  "nodeType": "map",
  "choices": [],
  "isEnd": false,
  "sortOrder": 4
}
```

分支节点示例：

```json
{
  "id": "choose_interest",
  "text": "接下来想先去哪里？",
  "nodeType": "choice",
  "choices": [
    { "text": "去看看学习区", "targetNodeId": "study_profile_gate" },
    { "text": "先去运动场转转", "targetNodeId": "sport_north_scene" }
  ],
  "isEnd": false
}
```

前端已兼容数据库下划线字段 `location_id` / `node_type`，但正式 DTO 请输出驼峰字段 `locationId` / `nodeType` / `choices`。`backend/utils/dto.js` 已提供对应映射。

## 3. 剧情节点导入

正文、地点顺序、专业映射和每个地点的两页结构见：

`backend/docs/浏览校园剧情脚本.md`

导入规则：

1. 先查询 `stories.code='campus'` 的 `id`。
2. 删除该故事旧节点，或按 `node_key` 全量 `UPDATE`；不要保留旧 `n1` 至 `n_end` 的 6 页占位文案。
3. 每个地点插入相邻两页：`{prefix}_scene`、`{prefix}_map`。两页使用同一个 `location_id`；前者 `node_type='scene'`，后者 `node_type='map'`。
4. 地图页的 `dialogue_text` 可统一使用：`看这片闪烁的金色区域，它就是{地点}；从{相邻地标}出发，很快就能找到入口。`
5. 仅最后的 `campus_end` 设置 `is_end=1`；其余节点均为 `0`。
6. `sort_order` 只用于后台排序和存档摘要，分支跳转以 `targetNodeId` 为准。

SQL 结构示例：

```sql
INSERT INTO story_nodes
  (story_id, node_key, dialogue_text, speaker, character_image_url, bg_image_url,
   location_id, node_type, choices_json, is_end, sort_order)
VALUES
  (@campus_id, 'life_north_dorm_scene', '01 到 04 号楼离学生综合服务中心很近……', '学姐',
   '/images/character_xuejie.png', '/images/story_bg1.jpg',
   'north_dormitories', 'scene', NULL, 0, 3),
  (@campus_id, 'life_north_dorm_map', '看这片闪烁的金色区域，它就是北区宿舍……', '学姐',
   '/images/character_xuejie.png', '/images/story_bg1.jpg',
   'north_dormitories', 'map', NULL, 0, 4);
```

## 4. 分支与专业路线

### 固定分支

`choose_interest`：学习区 / 运动区。

运动区结束时提供：`再去看看学习区` → `study_profile_gate`，`结束本次浏览` → `campus_end`。

学习区结束时提供：`再去运动场转转` → `sport_north_scene`，`结束本次浏览` → `campus_end`。

### 专业为空

`study_profile_gate` 必须提供：

```json
[
  { "text": "去填写专业", "targetNodeId": "study_profile_gate", "requiresMajor": true },
  { "text": "先看通用学习路线", "targetNodeId": "study_library_scene" }
]
```

前端发现 `requiresMajor=true` 且个人资料 `major` 为空时，会提示并进入个人资料页；不会错误地展示某个学院的楼宇。

### 专业匹配

请在用户资料中优先保存规范的 `college`；`major` 用于“是否已填写”和关键词兜底。专业分支的选项可附加 `targetByCollege`，前端将以 `college + major` 文本匹配第一个关键字并跳到对应节点；未匹配时回退 `targetNodeId`。

```json
{
  "text": "按我的专业继续",
  "targetNodeId": "study_generic_scene",
  "requiresMajor": true,
  "targetByCollege": {
    "机械与能源": "major_mechanical_scene",
    "材料": "major_material_scene",
    "人工智能": "major_ai_scene",
    "网络空间安全": "major_cyber_scene"
  }
}
```

完整关键字与目标节点按 `浏览校园剧情脚本.md` 的学院—地点表填写。不需要接口额外传 `userId`，鉴权仍只使用 `Authorization: Bearer <token>`。

## 5. 地点 ID 清单

地点 ID 的唯一来源是 `frontend/utils/campus-map.js`。剧情使用的主要 ID 包括：

`campus_north_gate`、`campus_east_gate`、`campus_south_gate`、`campus_west_gate`、`north_dormitories`、`northeast_dormitories`、`east_dormitories`、`central_dormitories`、`south_dormitories`、`student_service_center`、`olympic_restaurant`、`food_court`、`express_station`、`school_hospital`、`north_sports_ground`、`south_sports_ground`、`basketball_volleyball_courts`、`tennis_courts`、`olympic_venue`、`table_tennis_hall`、`swimming_pool`、`underground_training_hall`、`library`、`no1_teaching_building`、`no2_teaching_building`、`no3_teaching_building`、`no4_teaching_building`，以及剧情脚本列出的全部学院楼 ID。

## 6. 联调验收

1. 使用带 `campus` 剧情数据的后端地址登录小程序。
2. 检查 `GET /api/stories/campus/nodes`：不再是旧版 6 节点，且每个地点有连续 `scene`、`map` 两页。
3. 到分支页选择学习区、运动区，确认跳转到各自目标节点而非顺序播放。
4. 清空个人资料的 `major`，选择“去填写专业”，确认跳转资料页；填写后重新进入剧情，确认可进入专属路线。
5. 点击“地点地图”，确认地图聚焦地点且色块闪烁。
6. 完成剧情两次：首次显示新勋章，第二次不再显示“获得勋章”。
