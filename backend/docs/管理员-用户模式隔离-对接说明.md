# 管理员 / 用户模式隔离 —— 对接说明

> 负责人：任晟达（后端 B）
> 对接：栗守铭 / 张炜晨（前端，"我的"页入口）、黄懿卿（后端 A，可选加固）
> 目的：普通用户完全看不到管理员入口；只有白名单内的微信用户才在"我的"页看到"数据管理"入口。

---

## 一、安全模型（先看这个，别只看接口）

隔离分三层，**只有第 3 层是真正的安全边界**：

1. **藏入口（UI 层）**：前端只对白名单用户显示"数据管理"入口。——降低被发现/误触的概率，不是安全边界。
2. **可见性判定（本次新增的后端接口）**：`GET /api/admin/eligibility` 按当前登录用户的 `openid` 是否在白名单，返回 `canAdmin`。——**只决定 UI 显不显示，不下发任何权限**。
3. **资源鉴权（真正的安全边界，早已存在）**：每个 `/api/admin/*` 接口都由 `adminAuth` 中间件校验 adminToken（管理员密码登录换取）。普通用户拿 userToken 调后台 → `401 + 1003`。

> ⚠ 关键认识：**"隐藏入口"永远不是安全。** 任何人都能手写 HTTP 请求直接打 `/api/admin/stats`，前端藏没藏按钮跟这个请求无关。真正拦住越权的是第 3 层。本方案安全 = 白名单只管"藏"、密码/adminToken 只管"发权限"，两者分离。
>
> 已实测验证：白名单用户 `canAdmin=true`，但用他的 userToken 打 `/api/admin/stats` 仍返回 `401 + 1003`。

## 二、后端已实现（无需再改后端即可对接）

### `GET /api/admin/eligibility`

- **鉴权**：用户 token（`Authorization: Bearer <userToken>`，普通登录用户即可）。
- **响应**：
  ```json
  { "code": 0, "message": "success", "data": { "canAdmin": true } }
  ```
- 未登录 → `401 + 1002`；token 无效/过期 → `401 + 1003`。

### 配置白名单（后端 `.env`）

```
# 逗号分隔的 openid 列表，命中者才 canAdmin=true；留空 = 无人可见（安全默认）
ADMIN_OPENID_WHITELIST=openid_a,openid_b
```

- 改 `.env` 后**重启后端**生效（node 常驻进程不会自动重载）。
- **正式上线（MOCK_WECHAT=false）**：填组员真实微信 openid。获取办法：组员先登录一次，再查库
  `SELECT id,name,openid FROM users WHERE name='张三';` 把 openid 填进白名单。
- **本地联调（MOCK_WECHAT=true）**：openid = `mock_<code>`。用**固定** code 登录（如 code=`admin_rsd` → openid=`mock_admin_rsd`），把 `mock_admin_rsd` 填进白名单即可自测。
  （注意：mock 模式下每次 `wx.login` 的 code 不同会得到不同 openid，自测请固定 code。）

## 三、前端对接（"我的"页）

### 1. `utils/api.js` 加一个封装

```js
// 查询当前用户是否可见管理员入口（仅 UI 开关，不代表已有权限）
const checkAdminEligibility = () => request('GET', '/api/admin/eligibility');
// 记得 export
```

### 2. "我的"页 `my.js` onShow / onLoad 里调用

```js
async loadAdminEntry() {
  try {
    const { canAdmin } = await api.checkAdminEligibility();
    this.setData({ showAdminEntry: !!canAdmin });
  } catch (e) {
    // 未登录/失败一律不显示入口，安全默认
    this.setData({ showAdminEntry: false });
  }
}
```

### 3. `my.wxml` 用 `wx:if` 控制入口

```xml
<view wx:if="{{showAdminEntry}}" bindtap="goAdmin">🛠 数据管理</view>
```
点击 `goAdmin` → `wx.navigateTo({ url: '/pages/admin-login/admin-login' })`。
**进后台仍走现有管理员登录（密码换 adminToken），流程不变。**

> 备用入口（可选）：生成指向 `/pages/admin-login/admin-login` 的内部小程序码只发组员。
> 但后端仍靠密码 + adminToken 校验，扫码进来的人没密码照样进不去。

   保持这条红线即可。正式上线 `MOCK_WECHAT=false` 后 openid 由微信服务端换取，不可伪造。

## 五、可选加固（建议，需后端 A 黄懿卿）
