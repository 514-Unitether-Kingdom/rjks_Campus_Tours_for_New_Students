## 后端需要处理的内容

> 所有 SQL 均为 UTF-8。请使用支持 UTF-8 的 MySQL 客户端，并以实际数据库名称、账号执行；不要把开发环境临时公网地址写入前端代码。

1. **部署/刷新浏览校园剧情**：执行 `backend/sql/t9_campus_story_seed.sql`。
   - 它会建立/更新长剧情所需字段和节点，包括 `location_id`、`node_type`、`choices_json`、`grants_badge` 的使用内容。
   - 若线上已部署过旧版且只需修正子任务勋章时机，可执行 `backend/sql/t9_campus_badge_timing_migration.sql`，无需重置剧情节点。

2. **刷新医保报销剧情**：执行 `backend/sql/t10_medical_story_refresh.sql`。
   - 此脚本会删除并重建 `medical` 故事的节点，以写入新版 23 节点、多分支内容。
   - 请从 `backend/sql` 目录运行，因为脚本使用了同目录下的 `SOURCE t7_seed.sql`。
   - 已有的医保短剧情进度会被重置到开头（被删除节点的进度引用会置空）；已获得的勋章不会被删除。

3. **接口返回约定**：`GET /api/stories/:storyId/nodes` 的每个节点应返回（或可被 DTO 映射为）以下字段：
   - `id`：稳定的节点标识；选项的 `targetNodeId` 必须与它完全一致。
   - `text`、`character`、`bg`、`isEnd`。
   - 可选的 `locationId` / `location_id`、`nodeType` / `node_type`。
   - 可选的 `choices`：数组格式 `[{ "text": "选项文案", "targetNodeId": "目标节点 id" }]`。数据库字段 `choices_json` 需要正确解析为数组。

## 测试清单

### A. 通用前置条件

1. 使用一个可登录的测试账号；确认小程序已填入当前后端提供的调试地址（仅本地调试，不提交临时地址）。
2. 后端已执行上述剧情 SQL，`/api/stories/campus/nodes` 和 `/api/stories/medical/nodes` 能返回节点。
3. 建议准备两个账号：账号 A 用于首次通关/勋章测试，账号 B 用于重复进入/重复勋章测试。也可由后端清除账号 A 的相关进度与勋章后复测。

### B. 「浏览校园」测试

1. 从故事模式进入「浏览校园」，检查欢迎语单独占第一页；下一页才出现校门总览。
2. 依次进入三张校门地图页，确认地图自动弹出，且每页仅高亮以下成对地点：
   - 西门 + 西侧小门；
   - 北门 + 东门；
   - 南门 + 东南门。
   手动关闭地图后，剧情仍可继续；重复打开「地点地图」也应正常。
3. 逐步浏览每一个有地图的剧情页：高亮框应贴合目标建筑或场地，地图可缩放显示，未出现空白、错位或加载失败。
4. 在专业信息已填写的账号上，分别验证文档中涉及的专业分支会进入相应地点介绍；使用未填写专业的账号，验证出现填写提示且不会进入错误路线。
5. 生活区分支：依次浏览食堂、快递站、校医院等节点；**进入分支时不应获得勋章**，到生活区完成节点才应弹出一次新勋章提示。
6. 运动区分支：依次浏览场地、开放时间等节点；同样只在运动区完成节点首次获得勋章。
7. 对同一账号再次完成已获得勋章的分区：记录应可完成，但不应再次弹出「获得勋章」。
8. 从任意位置点击「上一页」，验证回到实际刚浏览的上一节点；特别检查经过分支选择后不会跳到不相关剧情。
9. 打开「探索地图」，确认底图是清晰的 `map2.png`；医保入口图标在校医院位置，点击确认后能进入医保报销流程。

### C. 「医保报销流程」测试

1. 从探索地图点击医保标记，进入后确认第一至第三页为校医引导、适用对象和通用规则，校医院地点地图可正常打开。
2. 到「现在的情况更接近哪一种」选择页，验证轻触剧情空白区域**不会**越过选择；必须点选选项才跳转。
3. 分别完整走完四条主路线：
   - 工作日普通门诊 → 挂号 → 校医院转诊 → 友谊医院就诊；
   - 周末/节假日/假期普通门诊 → 北京友谊医院；
   - 周末/节假日/假期普通门诊 → 离京急诊；
   - 突发急症；
   - 住院 → 分别选择「了解住院借款」与「不需要借款」。
4. 每条路线末尾都测试两个选择：
   - 「再了解其他就医情形」应回到四类情况总选择；
   - 「继续准备……材料」应进入正确的普通门诊、急诊或住院材料页。
5. 三类材料页均测试：可回到总选择，或进入提交、审核、到账和结束页；内容应与路线相符。
6. 在任意分支中连续点「上一页」，确认按真实浏览轨迹回退，而非按数据库排序回退；回到首节点时应提示已是第一页。
7. 首次完成：应调用完成接口并获得「医疗达人」勋章；再次完成：能正常返回，但不重复弹出新勋章提示。
8. 网络异常测试：断开/更换为无效地址后进入剧情，应显示加载失败提示；恢复有效地址后可重新进入。
