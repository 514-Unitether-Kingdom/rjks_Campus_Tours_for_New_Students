## 一、项目简介

### 1.1 产品定位

面向北京工业大学新生，通过**剧情互动**和**轻量游戏化机制**，帮助新生以沉浸式、趣味化的方式熟悉校园环境与常见办事流程。

### 1.2 核心功能

| 功能 | 说明 |
| --- | --- |
| 微信授权登录 | 用户通过微信授权完成身份认证 |
| 完善个人信息 | 采集姓名、性别、年级、学院（选择器）、专业（选择器） |
| 老用户自动登录 | 已注册用户再次进入直接跳转主页，无需重复填写信息 |
| 浏览校园剧情（长故事） | 通过点击推进剧情，了解校园建筑与场所，支持存档 |
| 办事流程剧情（短故事） | 通过剧情互动了解医保报销等办事流程，不支持存档 |
| 勋章系统 | 完成剧情自动发放勋章，已获得勋章彩色显示，未获得灰色显示 |
| 存档/读档 | 长故事支持进度保存和载入（最多5个存档） |
| 校园地图 | 查看平乐园校区平面图 |
| 管理员后台 | 查看注册用户、导出数据、预览剧情 |
| 登出功能 | 用户可主动退出登录，切换账号 |

### 1.3 成功标准

10人完成注册并使用。

## 五、功能模块说明

### 5.1 用户流程

#### 新用户首次使用

```text
进入小程序 → 微信授权登录 → 填写个人信息 → 保存 → 进入主页面
```

- 姓名、性别为必填项
- 学院、专业为**选择器**（下拉选择），专业列表根据所选学院动态变化
- 年级为选择器

#### 老用户再次登录

```text
进入小程序 → 检测到已注册 → 直接进入主页面（跳过授权弹窗和个人信息页）
```

- 通过本地缓存 `hasUserInfo` 和 `userInfo` 判断是否已注册
- 登录页自动跳转，并显示 "欢迎回来，XXX"

#### 登出

```text
主页面 → 我的 → 退出登录 → 确认 → 清除缓存 → 回到登录页
```

- 清除 token、userInfo、hasUserInfo、adminToken
- 勋章和存档数据保留（按 userId 区分）

### 5.2 长故事体验（浏览校园）

```text
主页面 → 故事模式 → 浏览校园剧情体验 → 点击推进剧情 → 
（可选）存档 → 到达最后一页 → 点击完成 → 获得勋章弹窗 → 返回故事模式
```

**特点：**

- 支持存档（最多5个）
- 支持查看历史对话
- 完成剧情自动发放勋章
- **已完成剧情再次进入**：显示"✅ 剧情已完成（勋章已获得）"

### 5.3 短故事体验（医保报销等）

```text
主页面 → 故事模式 → 办事流程剧情体验 → 
点击地图标记 → 进入剧情 → 点击推进 → 到达最后一页 → 
点击完成 → 获得勋章弹窗 → 返回
```

**特点：**

- 不支持存档
- 完成后地图标记由感叹号变为勾
- 完成剧情自动发放勋章

### 5.4 勋章系统

```text
主页面 → 故事模式 → 查看勋章
```

**显示规则：**

| 状态 | 图标 | 边框 | 文字 |
| --- | --- | --- | --- |
| 已获得 | 彩色勋章图标 | 金色边框 + 发光效果 | ✅ 已获得 |
| 未获得 | 灰色占位图 | 灰色调 | 🔒 未获得 |

**勋章列表：**

| 勋章ID | 名称 | 解锁条件 |
| --- | --- | --- |
| `badge_campus` | 🏛️ 校园探索者 | 完成浏览校园剧情 |
| `badge_medical` | 🏥 医疗达人 | 完成医保报销剧情 |
| `badge_card` | 💳 补办高手 | 完成一卡通补办剧情（后续迭代） |
| `badge_print` | 🖨️ 打印能手 | 完成打印流程剧情（后续迭代） |

### 5.5 存档系统

```text
主页面 → 故事模式 → 查看存档
```

- 显示所有存档列表（存档时间、进度）
- 支持载入存档
- 支持删除存档

## 七、数据存储说明

### 7.1 存储方式

当前所有数据使用微信小程序**本地缓存**（`wx.setStorageSync` / `wx.getStorageSync`）存储。

### 7.2 缓存Key说明

| 缓存Key | 存储内容 | 示例值 |
| --- | --- | --- |
| `token` | 用户登录令牌 | `mock_token_xxx` |
| `hasUserInfo` | 是否已注册（用于老用户判断） | `true` |
| `userInfo` | 当前用户个人信息 | `{ name, gender, grade, college, major }` |
| `badges_{userId}` | 已获得的勋章ID列表 | `['badge_campus']` |
| `completed_{userId}` | 已完成的剧情ID列表 | `['campus', 'medical']` |
| `completedProcesses` | 办事流程完成状态 | `{ medical: true }` |
| `saves_{userId}` | 存档列表 | `[{ slotId, storyId, nodeId, saveTime }]` |
| `adminToken` | 管理员登录令牌 | `mock_admin_token_xxx` |

### 7.3 登出时清除的缓存

| 缓存Key | 是否清除 | 说明 |
| --- | --- | --- |
| `token` | ✅ 清除 | 登录令牌 |
| `userInfo` | ✅ 清除 | 用户信息 |
| `hasUserInfo` | ✅ 清除 | 登录状态标记 |
| `adminToken` | ✅ 清除 | 管理员令牌 |
| `badges_{userId}` | ❌ 保留 | 按userId区分，切换账号保留 |
| `completed_{userId}` | ❌ 保留 | 按userId区分 |
| `saves_{userId}` | ❌ 保留 | 按userId区分 |

## 十一、已知问题与优化方向

### 11.1 已知问题

| 问题 | 影响 | 解决方案 |
| --- | --- | --- |
| 勋章数据使用本地缓存 | 换设备或清除缓存后勋章丢失 | 接入后端API持久化存储 |
| 图片资源缺失 | 部分页面显示占位或空白 | 添加真实图片资源 |
| 剧情内容为模拟数据 | 需要替换为真实校园内容 | 撰写真实剧情脚本 |
| 地图双指缩放 | 电脑端无法测试 | 需在真机上测试（电脑端鼠标不支持双指） |

### 11.2 优化方向

| 优先级 | 优化项 | 说明 |
| --- | --- | --- |
| 高 | 接入真实后端API | 用户数据、勋章数据、剧情数据持久化 |
| 高 | 替换真实剧情内容 | 基于学生真实经历撰写 |
| 高 | 添加真实校园地图 | 使用学校官方地图 |
| 中 | 完善UI设计 | 替换为最终设计稿 |
| 中 | 真机测试 | 在iOS和Android设备上测试 |
| 低 | 添加更多剧情 | 校园一天作息、打印流程等 |

### 11.3 测试注意事项

| 测试项 | 说明 |
| --- | --- |
| 地图缩放/拖拽 | 电脑端模拟器不支持双指缩放，需在**真机**上测试 |
| 勋章发放 | 完成剧情后需点击最后一页触发完成 |
| 老用户登录 | 清除缓存后需重新走新用户流程 |

### Q1: 打开项目后页面全白？

**A:** 检查以下几点：

1. 确保 `app.json` 中的页面路径正确
2. 点击「工具」→「清除缓存」→「清除全部缓存」
3. 查看控制台是否有语法错误
4. 重新编译

### Q2: 图片加载失败？

**A:** 在 `images/` 文件夹中添加对应图片，或临时使用网络占位图。

### Q3: 管理员登录失败？

**A:** 使用默认账号 `admin` / `123456`。如果还是失败，检查 `utils/api.js` 中的 `adminLogin` 函数。

### Q4: 勋章没有保存？

**A:** 当前版本使用本地缓存，清除缓存后会丢失。正式版本需要接入后端API。

### Q5: 如何真机预览？

**A:** 微信开发者工具中点击「预览」→ 扫码即可在手机上查看。

### Q6: 地图双指缩放怎么测试？

**A:** 电脑端模拟器不支持双指手势，需要在**手机真机**上测试。点击工具栏的「预览」→ 手机扫码打开，用双指在手机屏幕上缩放测试。

### Q7: 完成剧情后没有弹出勋章？

**A:**

1. 确保点击到了最后一页（显示"点击完成剧情"）
2. 查看控制台是否有报错
3. 检查 `utils/api.js` 中的 `completeStory` 函数是否正常

### Q8: 老用户为什么还是跳转到个人信息页？

**A:**

1. 检查本地缓存是否有 `hasUserInfo: true`
2. 检查 `userInfo` 是否完整
3. 清除缓存后重新登录一次
